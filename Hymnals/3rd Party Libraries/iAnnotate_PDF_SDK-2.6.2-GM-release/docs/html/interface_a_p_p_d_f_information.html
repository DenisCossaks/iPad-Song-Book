Detailed Description

Provides information about a PDF file. This information is cached in a disk file, which is available in the path attribute. PDF Information files are generated by the APPDFProcessor driver class, and are also used as an on-disk cache for user annotations.

- (BOOL) hasCharactersNotInSet: (NSCharacterSet *)  set
inPageRange: (NSRange)  pageRange 

Can be used to determine if there is any selectable text on a single page of the PDF based on a caller's specific criteria. Similar to hasTextOnPage:, but allows the caller to specify exactly what text should be excluded. For example, some callers might want to users the inversion of the alphanumericCharacterSet. Other users might not want to include the symbolCharacterSet and punctuationCharacterSet before inverting.

This method is can be used for checking if markup annotation tools (such as highlighter) and text selection functionality should be enabled.

Parameters:
setA set of characters that do not qualify as selectable.
pageRangeA range of 0-based page indices on which to check for characters not in set.
Returns:
YES if any characters not in set are detected on the specified page of the PDF document. Note that some PDF files that appear to be textual may not have embedded text; for example, if a scanned document has not been OCR'd.

- (BOOL) hasText

Used to determine if the PDF has any available text. If the PDF document has not yet been processed, this will still return YES if any characters have been discovered via live processing the currently-active page range (unless the APPDFViewOptions::liveProcessingDisabled is turned on).

This method will return YES whenever -allText will return a non-empty array. This means that this method will return YES if *any* characters are found in the document, even if they may not be visible in all contexts . Some PDFs include "hidden" text in various ways and for various reasons (for example accessibility). For this reason, this method may or may not be appropriate for deciding whether markup annotation tools (such as highlighter) and text selection functionality should be enabled. If a more restrictive set of criteria is needed, see the -hasCharactersNotInSet:inPageRange: method.

Furthermore, this alone should not be used to enable search functionality; that requires that the PDF is processed, so APPDFInformation::isProcessed must also be YES in this case. However, we recommend using the new (as of 2.4) isSearchable method instead.

Returns:
YES if any text was detected in the PDF document. Note that some PDF files that appear to be textual may not have embedded text; for example, if a scanned document has not been OCR'd.

Used to determine if there is any text embedded withinin a single page of the PDF. Similar to hasText, but restricts the query to the indicated page. If the PDF document has not yet been processed, this will still return YES if any characters have been discovered via live processing the specified page range (unless the APPDFViewOptions::liveProcessingDisabled is turned on).

This method will return YES whenever -textOnPage: will return a non-empty array. This means that this method will return YES if *any* characters are specified for the page, even if they may not be visible in all contexts . For this reason, this method may or may not be appropriate for deciding whether markup annotation tools (such as highlighter) and text selection functionality should be enabled. If a more restrictive set of criteria is needed, see the -hasCharactersNotInSet:inPageRange: method.

Furthermore, this alone should not be used to enable search functionality for a single page; that requires that the PDF is processed, so APPDFInformation::isProcessed * must also be YES in this case.

- (BOOL) importAnnotationXMLFromStream: (NSInputStream *)  inputStream

Used to import annotations in XML format into this information file. Please note that it is up to the client to make sure that the imported annotation data are valid -- and, in general, annotations will only be accurately placed on the same document from whence they originated. Importing annotations from a different base document can cause problems and even crashes (for example, if an annotation is on a page index that is too large for the host document).

This is primarily intended to be used to offer external storage of annotations (eg: on a remote server, or encrypted store) outside the document.

The XML format must match the format generated by "exportAnnotationXMLToStream:", provided as a single XML annotation or a sequence of tags inside of an "annotations" parent tag.

Parameters:
inputStreamThe NSInputStream from which the XML annotation data will be read.
Returns:
YES if the import succeeded.

- (id) initWithRawDatabaseHandle: (sqlite3 *)  handle

Initializes the PDF Information using a provided sqlite3 handle. The handle *must* remain valid for the entire lifetime of this object; note that the object may live after you've released a reference to it. Do not use the handle after passing it to this initializer; this handle must only be used by the APPDFInformation object once it is passed. If you need simulatenous access to the database, create a new handle for other uses.

This is not recommended unless you require it to work with a custom sqlite3 implementation. The full sqlite3 API must be properly supported by this custom implementation.

Used to perform a full-text search of the document. The parameter is an APSearch helper object, which includes an APSearchRequest object describing the search query and specifying various search options, and an APPDFSearchDelegate object which is used to return search result data.

A search operation can take some time, especially on large documents, so it is highly recommended that this method only be called on a background thread.
